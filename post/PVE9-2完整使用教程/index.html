一、PVE 是什么

Proxmox Virtual Environment (PVE) 是一个开源的企业级虚拟化平台，基于 QEMU/KVM（全虚拟化虚拟机）和 LXC（轻量级容器）双引擎，提供统一的 Web 管理界面。它支持：

特性 说明
双虚拟化引擎 同时管理 KVM 虚拟机和 LXC 容器
内置 WebUI 无需安装第三方管理工具，浏览器直接管理
集群管理 多节点集群，统一管理面板
高可用 (HA) 自动故障转移，保障业务连续性
SDN 软件定义网络，支持 VXLAN、EVPN、WireGuard
Ceph 集成 超融合分布式存储
ZFS 支持 高级文件系统，支持压缩、去重、快照
备份与恢复 内置备份管理器，支持全量/增量/定期备份

二、系统要求

最低配置

组件 要求
CPU 64 位 x86，支持 Intel VT-x / AMD-V
内存 2 GB（不含虚拟机）
磁盘 32 GB
网络 一张网卡

推荐配置（生产环境）

组件 推荐
CPU 多核处理器，支持 VT-d / AMD IOMMU
内存 16 GB+
磁盘 SSD + HDD 组合，或全闪存
网络 双网卡（管理 + 业务）或更多
RAID 硬件 RAID 或 ZFS 软 RAID

支持 Secure Boot（PVE 8.1+ 已支持），安装前可在 BIOS 中关闭或保持开启。

四、安装 PVE 9.2

启动安装程序

  1. 插入 USB 启动盘，从 USB 引导
  2. 出现 GRUB 菜单，选择 **”Install Proxmox VE (Graphical)”**（图形安装）
  3. 若显卡驱动异常，可选择 “Terminal UI” 终端模式

安装步骤

第 1 步：接受 EULA

阅读并接受最终用户许可协议。

第 2 步：选择目标磁盘

  • 选择安装磁盘
  • 点击 Options 可切换文件系统（ext4 / xfs / ZFS / btrfs）
文件系统 适用场景
ext4 + LVM 通用场景，稳定可靠
xfs + LVM 大文件性能更优
ZFS 需要快照、压缩、去重、RAID
btrfs 技术预览

第 3 步：选择地区与键盘布局

  • Location：选择 Asia/Shanghai 或离你最近的地区
  • Keyboard Layout：U.S. English（中文用户推荐）

第 4 步：设置密码与邮箱

第 5 步：网络配置

字段 说明 示例
Management Interface 管理网口 enp2s0
Hostname 主机名（FQDN） pve1.example.com
IP Address 管理 IP 192.168.1.100
Netmask 子网掩码 24
Gateway 默认网关 192.168.1.1
DNS Server DNS 服务器 8.8.8.8

注意： 主机名必须是 FQDN 格式（如 pve1.local），否则部分功能会异常。

第 6 步：确认安装

确认摘要信息无误后，点击 Install 开始安装，等待约 5-10 分钟。

安装完成后移除 U 盘，系统自动重启。

六、WebUI 概览

登录后界面分为以下几个核心区域：

区域 说明
左侧树 数据中心 → 节点 → 存储 → 虚拟机/容器

七、创建第一台虚拟机

准备工作：上传 ISO

  1. 在左侧选择你的节点 → 点击 local (pve1) 存储
  2. 切换到 ISO Images 标签页
  3. 点击 Upload 上传系统 ISO，或使用 URL 下载

常用镜像源：

创建虚拟机

  1. 点击右上角 Create VM
  2. General 页：
    • VM ID：自动生成或手动指定
    • Name：自定义名称
    • Start at boot：按需勾选
  3. OS 页：
    • ISO image：选择已上传的 ISO
    • Type：Linux / Microsoft Windows / Other
  4. System 页：
    • Graphics card：SPICE 或 VirtIO-GPU
    • BIOS：OVMF (UEFI)（推荐，支持 Secure Boot）
    • SCSI Controller：VirtIO SCSI single
    • Qemu Agent：勾选
  5. Disks 页：
    • Bus/Device：VirtIO Block（性能最好）
    • Storage：选择存储位置
    • Disk size：按需设置
    • Cache：Write back (unsafe) 可提升性能
  6. CPU 页：
    • Type：host（直通宿主 CPU 特性）
    • Cores：按需分配
  7. Memory 页：
    • Memory (MiB)：按需分配
    • Ballooning Device：勾选（动态内存调整）
  8. Network 页：
    • Bridge：vmbr0
    • Model：VirtIO (paravirtualized)
  9. Confirm 页确认，点击 Finish

安装系统

  1. 在 VM 列表中右键 → Start
  2. 点击 Console 进入 VNC 控制台
  3. 按常规方式安装操作系统
  4. 安装完成后安装 QEMU Guest Agent：

安装 QEMU Guest Agent 后，WebUI 可显示 VM 的 IP、内存使用等信息，并支持优雅关机。

安装 VirtIO 驱动（Windows）

  1. 下载 VirtIO ISO
  2. 上传到 PVE 存储
  3. 在 VM 的 Hardware 中添加 CD/DVD 驱动，挂载 VirtIO ISO
  4. 进入 Windows，运行 virtio-win-guest-tools.exe 安装所有驱动

八、创建 LXC 容器

PVE 的 LXC 容器比虚拟机更轻量，共享宿主机内核，启动只需秒级。

下载容器模板

  1. 左侧选择节点 → 展开 local (pve1) 存储
  2. 切换到 CT Templates 标签页
  3. 点击 Templates，选择所需模板（如 ubuntu-24.04-standard）→ Download

也可以手动下载模板：

创建容器

  1. 点击 Create CT
  2. General：设置 CT ID、Hostname、密码或 SSH key
  3. Template：选择已下载的模板
  4. Root Disk：设置磁盘大小和存储位置
  5. CPU：分配核心数
  6. Memory：分配内存（可勾选 Swap / Ballooning）
  7. Network：
    • Bridge：vmbr0
    • IPv4：DHCP 或手动 Static
  8. DNS：默认使用宿主机 DNS
  9. 确认并 Finish

九、存储管理

PVE 支持多种存储后端，统一通过 Storage 管理。

支持的存储类型

类型 说明 适用场景
Directory 目录存储（本地文件系统） 单节点测试
LVM / LVM-thin 逻辑卷管理 高性能本地存储
ZFS 高级文件系统 需要快照、压缩、RAID
NFS 网络文件系统 共享存储
Ceph/RBD 分布式存储 超融合集群
Proxmox Backup Server 专用备份存储 异地备份
iSCSI 块存储 SAN 对接

添加存储

  1. 左侧选择 Datacenter → Storage
  2. 点击 Add，选择类型
  3. 填写配置（如 NFS 需填服务器 IP 和导出路径）

存储内容类型

Content 说明
VM images 虚拟机磁盘镜像
ISO images ISO 镜像
CT templates 容器模板
Backups 备份文件
Container 容器根目录
Snippets 脚本片段
Import 导入文件

十、网络配置

默认网络

安装时创建的 vmbr0 是 Linux Bridge，虚拟机/容器通过它连接物理网络。查看网络配置：

常见网络模式

模式 说明 配置方式
Linux Bridge 桥接到物理网卡 安装时自动创建 vmbr0
Linux Bond 多网卡绑定（冗余/负载均衡） 创建 bond，再创建 bridge
VLAN 网络隔离 Bridge 上启用 VLAN Aware
OVS Open vSwitch，高级 SDN 安装 openvswitch-switch
SDN PVE 9.2 原生 SDN Datacenter → SDN

配置 Bond 网卡

VLAN Aware Bridge

在 WebUI 中编辑 vmbr0，勾选 VLAN Aware，即可在 VM 网卡中指定 VLAN Tag，实现网络隔离。

十一、备份与还原

手动备份

在 VM/CT 上右键 → Backup → 选择存储位置，支持快照模式（无需停机）。

定时备份

  1. Datacenter → Backup
  2. 点击 Add，配置备份计划：
配置项 说明
Storage 备份存储位置
Schedule cron 表达式（如 0 3 * * * 凌晨 3 点）
Selection Mode 全选 / 排除 / 仅选
Mode Snapshot（推荐，不影响运行）
Compression Zstd（压缩率高）
Retention 保留策略（最后 N 天/周/月）

备份到 Proxmox Backup Server

PBS 是 PVE 生态的专用备份方案，支持数据去重和增量备份：

还原

  1. 左侧选择备份存储 → Backups
  2. 选中备份 → Restore

十二、集群与高可用

创建集群

  1. 在第一个节点上：Datacenter → Cluster → Create Cluster
  2. 输入集群名称
  3. 在其他节点上：Datacenter → Cluster → Join Cluster
  4. 输入第一个节点的 IP 和 root 密码

集群网络要求

  • 节点间延迟 < 2ms
  • 建议配置专用心跳网络（corosync）
  • NTP 时间必须同步

配置 HA（高可用）

  1. Datacenter → HA → Groups → 添加 HA 组，指定节点优先级
  2. Datacenter → HA → Resources → Add，选择要保护的 VM/CT

PVE 9.2 新增 动态负载均衡，HA Manager 可根据实时资源使用情况自动迁移虚拟机，实现集群负载均衡。

集群维护

PVE 9.2 新增 disarm-ha / arm-ha 命令，可在集群维护前安全解除 HA 仲裁，避免误触发节点隔离：

十三、PVE 9.2 新特性亮点

1. 动态负载均衡

CRS（Cluster Resource Scheduler）新增动态调度模式，基于实时节点和虚拟机的资源利用率，自动迁移 HA 托管的虚拟机以平衡集群负载。

2. SDN 重大升级

  • WireGuard 支持：加密隧道协议作为 SDN Fabric
  • BGP Fabric：基于 eBGP 无编号 Underlay
  • 路由映射与前缀列表：细粒度 BGP/EVPN 路由过滤
  • OSPF 路由重分发：多协议路由架构
  • EVPN IPv6 Underlay：全面支持 IPv6
  • 自定义 CPU 模型：在 WebUI 中创建和管理自定义 CPU 模型

3. HA 维护模式

disarm-ha / arm-ha 命令，在集群网络维护时安全暂停 HA 仲裁，完成后自动恢复。

4. Ceph Tentacle 20.2

新安装默认使用 Ceph Tentacle 20.2，同时支持 Ceph Squid 19.2.3。

5. Linux Kernel 7.0

基于最新的 7.0 内核，QEMU 11.0，LXC 7.0，ZFS 2.4。

6. 安装器增强

  • 支持 PXE/iPXE 自动安装
  • 支持 Embed 认证令牌
  • 支持引导时 Ctrl-C 进入调试 Shell

常见问题

Q1：安装后无法访问 WebUI？

PVE 默认安装 fail2ban，多次输错密码可能被临时封禁。

Q2：虚拟机无法联网？

  • 确认 VM 网卡桥接到 vmbr0
  • 检查物理网卡是否正常：ip a
  • 检查 PVE 防火墙配置：Datacenter → Firewall → Options → Enable: No

Q3：如何扩展现有磁盘？

Q4：PVE 经常弹出”无有效订阅”提示？

参考本文第五节中的去订阅弹窗脚本。

Q5：如何迁移虚拟机到另一台 PVE？

  • 在线迁移：右键 VM → Migrate（需集群环境）
  • 离线迁移：备份 → 还原到目标节点
  • 远程迁移：PVE 9.x 支持跨集群远程迁移

Q6：如何从 PVE 8.x 升级到 9.2？

注意： 跨大版本升级前务必做好完整备份，并在测试环境先行验证。

总结

Proxmox VE 9.2 作为最新的稳定版本，在虚拟化性能、SDN 网络、高可用调度和易用性方面都有显著提升。无论你是个人开发者搭建家庭实验室，还是运维人员部署生产环境，PVE 都是一个功能强大且完全开源的选择。

建议后续进一步学习：

  • Proxmox Datacenter Manager 多数据中心管理
